The Lifecycle of a Request

The server operates on a non-blocking I/O model to ensure that one slow client doesn't stall the entire pipeline.

1. Socket()   -> Create the endpoint
2. Bind()     -> Attach to port 8080
3. Listen()   -> Wait for incoming traffic
4. Accept()   -> Hand off to Thread Pool
5. Respond()  -> Send headers + data
